# Contexto · Facturación en Salud Colombia

Repositorio de conocimiento (vault de Obsidian) que centraliza el contexto para diseñar la arquitectura de una aplicación web de **facturación electrónica en salud** en Colombia, que maneja **información financiera y médica sensible**.

**Ábrelo con [Obsidian](https://obsidian.md)** apuntando a esta carpeta como *vault*. Empieza por `Inicio.md`.

## Estructura
- `00 · Meta` — cómo usar, convenciones, estado del proyecto
- `01 · Contexto del proyecto` — visión, alcance, actores, preguntas abiertas
- `02 · Fuentes` — fichas de fuente (Figma, documentos, Replit, oficiales, otras)
- `03 · Dominio y negocio` — conceptos, procesos, glosario
- `04 · Marco normativo` — normativa y cumplimiento
- `05 · Requisitos` — funcionales y no funcionales
- `06 · Modelo de datos` — entidades y diccionario
- `07 · Arquitectura` — vistas, componentes y decisiones (ADR)
- `08 · Base de datos` — diseño de la base de datos
- `09 · Flujos de información` — flujos de datos
- `_Plantillas`, `_Adjuntos`

Para el flujo de trabajo, ver `00 · Meta/Cómo usar este repositorio.md`.
